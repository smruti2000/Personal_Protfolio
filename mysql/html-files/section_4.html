SECTION-4

Inserting Data

  • Use the INSERT statement to enter data into a table.
  • For example:
    mysql> INSERT INTO pet VALUES ('Puffball','Diane','hamster','f', '1999-03-30',NULL);

NOT NULL

  • The NOT NULL constraint is a column constraint that ensures values stored in a column are not NULL.
  • For Example:
    mysql> CREATE TABLE pet (name VARCHAR(20) NOT NULL, owner VARCHAR(20) NOT NULL, species VARCHAR(20), sex CHAR(1) NOT NULL, birth DATE, death DATE);

DEFAULT

  • The MySQL DEFAULT keyword is a database CONSTRAINT or RULE that is applied when inserting new records into a table. If the value is not provided to any column while inserting a new row in the table, the default value will be used instead.
  • mysql> CREATE TABLE pet (name VARCHAR(20) DEFAULT 'SOME MESSAGE', owner_contact_no VARCHAR(20), species VARCHAR(20), sex CHAR(1), birth DATE, death DATE);

PRIMARY KEY

  • A primary key is a column or a set of columns that uniquely identifies each row in the table. ...
  • Any attempt to insert or update NULL to primary key columns will result in an error.
  • Note that MySQL implicitly adds a NOT NULL constraint to primary key columns
  • A table can have one an only one primary key.
  • For Example:
    mysql> CREATE TABLE pet (Pet_Id INT, name VARCHAR(20) DEFAULT 'SOME MESSAGE', owner_contact_no VARCHAR(20), species VARCHAR(20), sex CHAR(1), birth DATE, death DATE, primary key(Pet_Id));

Auto Increment

  • The "AUTO_INCREMENT" on the category_id field.
  • This causes the category Id to be automatically generated every time a new row is inserted into the table.
  • It is not supplied when inserting data into the table, MySQL generates it.
  • For Example:
    mysql> CREATE TABLE pet (Pet_Id INT AUTO_INCREMENT, name VARCHAR(20) DEFAULT 'SOME MESSAGE', owner_contact_no VARCHAR(20), species VARCHAR(20), sex CHAR(1), birth DATE, death DATE, primary key(Pet_Id));
